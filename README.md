# MikroTik Routes Uploader

GUI приложение для загрузки маршрутов в MikroTik из текстового файла.

## Возможности

- 🎨 Удобный графический интерфейс (GUI)
- ✅ Валидация IP-адресов и CIDR нотации
- 📋 Поддержка больших списков маршрутов
- 📝 Логирование всех операций
- 🔄 Многопоточная загрузка (интерфейс не зависает)
- ⚠️ Обработка ошибок с подробными сообщениями

## Требования

- Python 3.7+
- tkinter (обычно идёт с Python)
- routeros_api

## Установка

1. **Клонируем репозиторий:**
```bash
git clone https://github.com/ваш-юзер/mikrotik-routes-uploader.git
cd mikrotik-routes-uploader
```

2. **Устанавливаем зависимости:**
```bash
pip install -r requirements.txt
```

3. **Запускаем приложение:**
```bash
python upload_route_gui.py
```

## Использование

### Подготовка файла со списком сетей

Создайте текстовый файл (например `networks.txt`) со списком сетей в формате CIDR:

```
89.113.122.0/24
10.0.0.0/8
192.168.0.0/16
# Строки, начинающиеся с #, игнорируются
172.16.0.0/12
```

**Поддерживаемые форматы:**
- CIDR нотация: `192.168.1.0/24` ✓
- Отдельные IP: `8.8.8.8/32` ✓
- Пустые строки и комментарии - игнорируются ✓

### Загрузка маршрутов

1. Запустите приложение
2. Заполните поля подключения:
   - **Router IP** - IP адрес MikroTik
   - **Username** - имя пользователя
   - **Password** - пароль
   - **Gateway** - IP адрес шлюза (по умолчанию `192.168.27.250`)
   - **Comment** - опциональный комментарий к маршрутам
3. Выберите файл со списком сетей кнопкой "Обзор..."
4. Кликните "Загрузить маршруты"
5. Подтвердите действие

## Валидация данных

Приложение проверяет:
- ✓ Формат IP-адреса для маршрутизатора и шлюза
- ✓ CIDR нотацию для каждой сети
- ✓ Существование файла
- ✓ Наличие хотя бы одной валидной сети в файле

## Обработка ошибок

При возникновении ошибок:
1. Ошибка отображается в лог-секции приложения
2. Всплывающее окно с кратким описанием
3. Загрузка продолжается для остальных маршрутов (если возможно)

## FAQ

**Q: Можно ли загрузить маршруты без комментария?**  
A: Да, поле Comment опционально. Оставьте его пустым.

**Q: Что делать если соединение прерывается во время загрузки?**  
A: Приложение покажет ошибку. Проверьте сетевое соединение и попробуйте снова.

**Q: Можно ли отменить загруженные маршруты?**  
A: Нет встроенной функции. Удалите маршруты вручную через консоль MikroTik или RouterOS Interface.

**Q: Какой максимальный размер файла?**  
A: Ограничений нет, но очень большие файлы (>10000 маршрутов) могут загружаться долго.

## Решение проблем

### Ошибка: `No module named 'routeros_api'`
```bash
pip install routeros-api
```

### Ошибка подключения к MikroTik
- Проверьте IP адрес
- Проверьте имя пользователя и пароль
- Убедитесь, что пользователь имеет права на добавление маршрутов
- Проверьте сетевое соединение

### Интерфейс не отображается корректно
- Убедитесь, что установлена последняя версия Python
- Переустановите tkinter:
  ```bash
  # Ubuntu/Debian
  sudo apt-get install python3-tk
  
  # macOS
  brew install python-tk@3.11
  
  # Windows - переустановите Python с опцией tkinter
  ```
